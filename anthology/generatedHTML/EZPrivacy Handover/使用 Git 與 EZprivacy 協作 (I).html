使用 Git 與 EZprivacy 協作 (I)

Author: Chiawei

相信各位已經有安裝 Dropbox 與 Git 在電腦上了，我們接下來來示範如何提取 Git 資料夾中的資料。跟著本文章提供步驟做，可以從 Git Repository 中提取出 EZprivacy 專案內容，而下篇文章則會示範如何 commit 與 push 更新。

在這之前，請各位先參考欣達或我提供的連結，以學習一點基礎語法與概念：

  • 連猴子都能懂的 Git 入門指南
  • Git Book

從連結中各位將學到如何用以下指令在本地端使用 Git 來管理版本：

$ git init
$ git add
$ git commit 
$ git log
$ git diff
$ git branch

而本說明則會著重在與遠端 Git Repository 協作 的部份。

與遠端 Git Repository 協作

※ 利用 git clone 取得一份專案的 copy

現在在 Dropbox 中的那個資料夾是一個 bare repositry，也就是裏面只有 Git 的設定檔與資料庫，看不見專案的檔案。我們也不需要操作這些檔案結構。

那要怎麼從這個 bare repository 中提取出 EZprivacy 的專案呢？請在你的工作目錄（例如 Eclipse 的 workspace）開啟 Terminal，輸入如下指令：

$ git clone /<本地_Dropbox_資料夾的路徑>/vscclab_project/

便可以見到現在多出一個資料夾 vscclab_project，點進去看可以看到專案內容。我們現在要 cd 進去，並用 git status 來察看目前的狀態：

$ cd vscclab_project
$ git status

On branch master
Your branch is up-to-date with 'origin/master'.

nothing to commit, working directory clean

前兩句的意思是：我們現在在本地端的 master 分支上，而這個分支與遠端的 origin/master 分支指向同一個版本。

※ 切換到最新分支

雖然現在已經在 master 這個分支上，不過這個專案主要是在 develop 分支上進行的，請切換過去：

$ git checkout develop

Branch develop set up to track remote branch develop from origin.
Switched to a new branch 'develop'

這時檔案會有所更動。這個輸出結果的意思是：Git 產生了一個本地端的分支 develop 來指向遠端 origin/develop 所指的位置。使用 gitk --all 可以發現我們已經切換到比較新的版本了：

各位便可以開始改動專案內容。

其他提醒

  • Cluster組：現在這個資料夾已經是個 eclipse 專案，我們可以直接匯入到 Eclipse 中使用。原本 package 的檔案結構有點複雜，所以我們可以創一個 ezprivacy.cluster.* 的 package來放新的程式。

.

  • 登入頁面組：Eclipse 雖然可以寫 JSP，但是目前的檔案配置似乎不是依照 eclipse JSP 的方式來擺放的。你們可能需要開個新資料夾來放，或是一起來討論其他方法。
