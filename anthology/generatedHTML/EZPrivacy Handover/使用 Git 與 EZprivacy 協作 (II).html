看完第一篇文章之後，各位應該已經能夠從遠端 clone 一個資料夾下來更改或添加程式了，接下來將會介紹如何新增你的分支並 push 上去，或將別人寫好的程式 pull 下來。

本地端的暫存

首先，各位應該已經知道 add 與 commit 的用途了，假設我現在新增了一個檔案，我要將它暫存到暫存區，讓我日後提交，則我應該使用 git add：

$ git add --all

然後用 git status 可以看到新增的檔案會以綠色的字顯示，表示已經成功存入暫存區 (Staged)：

$ git status

On branch develop
Your branch is up-to-date with 'origin/develop'.

Changes to be committed:
(use "git reset HEAD <file>..." to unstage)

    new file:   "for developer/git/使用 Git 與 EZprivacy 協作 (I).md"

在本地端創造新的分支

接著我們就要提交版本了。不過由於人數眾多，如果大家都提交在同一個分支上會有同步版本問題，導致常常需要 merge，因此希望每個人（或每一組）先各自建立一個 branch，並切換過去那個新分支：

$ git branch ... 
$ git checkout document

例如當我建立一個新的 document 分支：

$ git branch document 
$ git checkout document

[A] "for developer/git/使用 Git 與 EZprivacy 協作 (I).md"
Switched to branch 'document'

用gitk 可以看到多出了一個新指標 document 指在我目前的進度位置

提交版本

接著我們就可以將版本提交（commit）在這個指標的位置：

$ git commit -m "add git tutorial"

[develop bd9b3fd] add git tutorial
  1 file changed, 13 insertions(+)
  create mode 100644 "for developer/git/使用 Git 與 EZprivacy 協作 (I).md"

表示我們已經成功提交新版本，該版本註解為 add git tutorial。我們用 gitk --all 來察看：

可以看到版本圖已經岔開，document 分支指著其中一端，表示本地端的提交已經完成了。

Push 到遠端資料夾

接下來我們要用 push 來將新版本 push 到我們的共用資料夾：

$ git push origin document:gitdoc 

Counting objects: 19, done.
Delta compression using up to 2 threads.
Compressing objects: 100% (5/5), done.
Writing objects: 100% (5/5), 2.10 KiB | 0 bytes/s, done.
Total 5 (delta 2), reused 0 (delta 0)
To /home/autodavid/Dropbox/vscclab_project/
 * [new branch]      document -> gitdoc

意思是 push 到預設的遠端資料夾（代號通常為 origin），要將本地的 document 分支 push 到遠端的 gitdoc 分支。由於遠端沒有這個分支，所以 git 會自動創一個。我們用 gitk --all 來看這現象：

可以看到遠端多了一個 gitdoc 分支，那就是遠端的新分支。

Pull

最後，如果你和別人待在同一個分支上，可以用 git pull 來同步這個分支的最新狀態。
